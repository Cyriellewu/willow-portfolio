The premise

Planning is not the same as living.

Most tools preserve the plan and discard the day. I wanted a quieter product: a place where a calendar, a small note, a location, or an unexpected conversation could remain as evidence of a life well-lived.

Public demo rendered with repository sample data. Nothing in this image belongs to a real account.

The system

One input, several useful forms of context.

I built the application as a React and TypeScript SPA with Supabase Auth, Postgres, and Edge Functions. Smart input classifies a fragment of daily life, while calendar and map views make it possible to revisit when and where a moment happened.

CapturePlan, note, focus session, or remembered moment
ClassifyAI-assisted input routed into a useful record
RevisitCalendar and place context, not a productivity score

Evidence

Product behavior is backed by a testable system.

The interface sits on a full-stack implementation rather than a static concept: authentication, database records, serverless workflows, and centralized design tokens. The project card records 51 passing Vitest tests across its core flows.

Privacy boundary

The public view is deliberately fictional.

This site shows only a local public demo using sample data. No map view, real account, private journal, or connected calendar is exposed here.
