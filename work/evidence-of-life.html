The premise

Capture context that conventional planning tools omit.

Most tools preserve the plan but discard the surrounding context. I built a system where a calendar, a short note, a location, or an unexpected event can remain connected to the day it happened.

The system

One input, several useful forms of context.

I built the application as a React and TypeScript SPA with Supabase Auth, Postgres, and Edge Functions. Smart input classifies a fragment of daily life, while calendar and map views make it possible to revisit when and where a moment happened.

Evidence

Product behavior is backed by a unit-tested implementation.

The interface sits on a full-stack implementation rather than a static concept: authentication, database records, serverless workflows, and centralized design tokens. The project records 51 passing Vitest tests across parsing, classification, time, and timer utilities.

Unresolved product question

How do you show continuity without turning it into pressure?

The product research pushes against guilt-driven streak mechanics, while the current implementation still records habit streaks. The next product question is how to preserve completion history without making it the source of motivation.

Privacy boundary

The public view is deliberately fictional.

This site shows only a local public demo using sample data. No map view, real account, private journal, or connected calendar is exposed here.
