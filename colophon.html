How this site works.

A living field notebook

This site is a working document, not a finished portfolio. It changes as projects develop, thinking sharpens, and new work becomes ready to share. Some pages are polished; others are honest first drafts.

What is published

  • Project case studies drawn from published papers, public repositories, and course work
  • Sample data and synthetic illustrations only — no real user data
  • Aggregate statistics (e.g. SSIM histograms) without source imagery
  • Public-safe career and project facts selected for this site

What is intentionally omitted

  • Biometric images or identifiers (palm-vein project uses synthetic illustrations)
  • Proprietary datasets, NDA-covered details, or private account data
  • Internal details of employer work beyond high-level public role descriptions
  • Personal lifestyle, hobbies, or media consumption not relevant to work

Technical stack

Static HTML, CSS, and vanilla JavaScript. No frameworks, no build step, no external scripts, no analytics, no cookies. Hosted on GitHub Pages.

Accessibility

  • Semantic HTML with landmark regions and ARIA labels where needed
  • Skip-to-content link on every page
  • Keyboard-navigable interactive elements
  • Respects prefers-reduced-motion
  • Color contrast targets WCAG AA against the sage/cream palette

If something isn't working well with your setup, I'd like to know — email me.

Update policy

I update this site when there is something worth saying, not on a schedule. Major project additions or corrections get a commit; small copy edits accumulate. The Now page carries a date so you know when it was last true.
